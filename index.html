Migration Tools

Editor tools for cleaning up missing scripts, broken references, and third-party Visual Scripting graphs when you move a Unity project from one platform to another.

Download the latest .unitypackage View on GitHub

Free and MIT licensed. Works in Unity 6.

Why these exist

When a 3D content platform shuts down, or drops the features you built on, you are often left with a Unity project full of that platform's proprietary scripts. They throw errors all over the console, and a published build can load to a black screen because of them. Cleaning that up by hand, object by object, is slow and easy to get wrong.

These tools find what's broken, mark it so you can see exactly where it was before you remove anything, and clear out the parts that only ever worked on the old platform. The point is to get you back to a project you can actually work in, without starting from scratch.

This is not a magic port. Proprietary hooks that a platform embedded throughout a project can't all be untangled automatically. What these tools do is clean up what's left and save you real time, and they get better as more projects come through them.

They come from the Unreality3D Creator Dashboard, which comes built in to the free Unreality3D template Unity project. This is the same set, packaged on its own so you can run them in any Unity project.

What's inside

See it first

Scan and highlight problems

Before you change anything, light up every problem object right in the Scene view, color coded so nothing is a guess. Red is a missing script, yellow is a missing reference, blue is a placeholder. One button also selects them all in the Hierarchy and tells you how many it found.

Missing scripts

Missing scripts

Replace missing scripts with visible placeholder components, so you can see where each one was before you decide what to do with it. When you're ready, remove the placeholders, or clear missing scripts straight out of your scenes and prefabs.

Missing references

Missing references

Find object references that went missing inside components that still exist, track them with a placeholder, and jump straight to them in the Hierarchy to rewire. Clear the placeholders when you're done.

Visual Scripting

Broken Visual Scripting graphs

Clear Visual Scripting graphs that are full of another platform's node types, the kind that cause deserialization errors and broken builds. It works on the open scene and across a whole folder of imported assets and prefabs in one pass, and it clears the broken graphs without deleting your GameObjects.

Set up for Spatial out of the box. Adding another platform is a one-line change in the code, using the prefix shown in the red "Script Missing" nodes.

Install

  1. Download the latest MigrationTools.unitypackage.
  2. In Unity, go to Assets > Import Package > Custom Package, pick the file, and import.
  3. Open the tools from Tools > Unreality3D > Migration Tools.
Works in Unity 6. Every tool can be undone with Ctrl+Z. The two Visual Scripting tools need Unity's Visual Scripting package (com.unity.visualscripting). Projects coming from platforms that used visual scripting usually already have it, and if it isn't installed, those two tools show a note while everything else still works.
